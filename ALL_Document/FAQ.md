# Email Parser 2.0 - FAQ (Часто задаваемые вопросы)

---

## 📖 Основные вопросы

### Q1: Что это за программа?
**A:** Email Parser 2.0 — это приложение для автоматического парсинга и анализа электронных писем. 
- Загружает письма из IMAP почты
- Извлекает контакты, компании, продукты
- Сохраняет результаты локально
- Предоставляет веб-интерфейс для просмотра

**Используется для**: CRM импорт, сбор контактов, обработка входящей корреспонденции.

---

### Q2: Как установить?
**A:** Полная инструкция в [INSTALLATION.md](INSTALLATION.md).

**Быстро (3 шага)**:
```bash
1. pip install -r requirements.txt
2. Отредактируйте config.yaml
3. python main.py web
```

**Время установки**: 5-10 минут

---

### Q3: Нужен ли интернет?
**A:** 
- 🟢 **Локальный анализ** (local-nn): НЕ требуется интернет
- 🟡 **Ollama**: НЕ требуется интернет
- 🔴 **IMAP парсинг**: Требуется для скачивания писем

**После первой загрузки моделей** (200MB для local-nn) интернет уже не требуется!

### Q15: Как безопасно хранить пароль?
**A:** НЕ храните пароль в `config.yaml` для production!

**Способ 1: Переменные окружения**
```bash
export IMAP_PASSWORD=your_password
```

**Способ 2: Файл .env**
```
IMAP_PASSWORD=your_password
```

**Способ 3: Password manager**
```bash
python main.py -password "$(pass show email-password)"
```

---

### Q16: Может ли программа отправлять письма?
**A:** **НЕТ**. Программа только **читает** письма.

Это функция:
- ✅ Чтение из IMAP
- ✅ Анализ текста
- ✅ Сохранение результатов

Это НЕ функция:
- ❌ Отправка писем
- ❌ Изменение писем
- ❌ Удаление писем

**Безопасность**: Все письма остаются на сервере нетронутыми.

---

### Q17: Как часто обновлять новые письма?
**A:** Зависит от частоты входящих писем:

```bash
# Ежедневно (рекомендуется)
python main.py reprocess --only-missing

# Или добавьте в cron (Linux/Mac):
0 9 * * * cd /path && python main.py reprocess --only-missing

# Или в Windows Task Scheduler
```

**Оптимально**: Каждый день в 9 утра или после рабочего дня.

## 🐛 Проблемы и решения

### Q18: Почему программа медленно обрабатывает?
**A:** Несколько причин:

| Причина | Решение |
|---------|---------|
| Слабый ПК | Используйте local-nn вместо ollama |
| Много писем | Разобрать на части, параллельная обработка |
| Нет памяти | Закройте другие приложения |
| HDD диск | Используйте SSD если возможно |
| Первый раз | Нормально, модель загружается впервые |

---

### Q19: Почему письмо анализировано неправильно?
**A:** Несколько причин:

1. **Письмо на английском**: local-nn может быть менее точен
   - Решение: Используйте `ollama` для лучшей точности

2. **Сложное форматирование**: HTML, MIME, вложения
   - Решение: Это нормально, алгоритм extracting работает

3. **Опечатки в исходном письме**
   - Это не ошибка программы

4. **Очень короткое письмо**
   - Решение: Нейросеть работает лучше с полным текстом

**TL;DR**: Это **автоматический** анализ (не 100% точность).
