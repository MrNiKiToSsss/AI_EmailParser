## Что это такое?

**Email Parser 2.0** - это приложение для автоматического парсинга и анализа электронных писем.

### Основные возможности:
- 📧 Извлечение писем из IMAP почтовых ящиков
- 🤖 Автоматический анализ текста с использованием нейросети
- 💾 Сохранение и переобработка писем
- 🌐 Веб-интерфейс для просмотра и поиска
- 📊 Группировка по компаниям и продуктам
- 📈 Экспорт в Excel
- ⚡ Высокая производительность (13+ писем/сек)

### Что программа извлекает из писем:
- 👤 Полные имена контактов
- 📧 Email адреса
- 📞 Номера телефонов
- 🏢 Названия компаний
- 💼 Должности
- 🛍️ Интересующие продукты

## Установка и подготовка

### Системные требования
- **ОС**: Windows 10+ / Linux / macOS
- **Python**: 3.9+
- **RAM**: 4GB minimum, 8GB+ рекомендуется
- **Место на диске**: 2GB для моделей + данные

### Шаг 1: Клонирование репозитория
```bash
git clone <repository-url>
cd email_parser20
```

### Шаг 2: Создание виртуального окружения
```bash
python -m venv ollama_env
```

**Активация (Windows)**:
```bash
ollama_env\Scripts\activate
```

**Активация (Linux/macOS)**:
```bash
source ollama_env/bin/activate
```

### Шаг 3: Установка зависимостей
```bash
pip install -r requirements.txt
```

### Шаг 4: Конфигурация
Отредактируйте `config.yaml`:

```yaml
imap:
  server: mail.nic.ru          # IMAP сервер
  port: 993                    # Порт (обычно 993 для IMAPS)
  username: your_email@mail.com # Ваша почта
  password: your_password       # Пароль
  mailbox: INBOX               # Папка (обычно INBOX)

storage:
  emails_dir: ./data/emails        # Папка для сохранения писем
  results_dir: ./data/results      # Папка для результатов анализа
  processed_db: ./data/processed.json  # БД обработанных писем

ollama:
  api_url: http://localhost:11434/api/generate  # API Ollama (если используется)
  model: mistral:7b                # Модель Ollama
```

### Шаг 5: Первый запуск
```bash
python main.py web
# Откройте http://localhost:5555
```

### 1. WEB - Веб-интерфейс

#### Описание
Запускает локальный Flask сервер для просмотра, поиска и экспорта результатов анализа писем.

#### Синтаксис
```bash
python main.py web [опции]
```

#### Опции
```
--host АДРЕС      В каком адресе слушать (по умолчанию: 0.0.0.0)
--port ПОРТ       На каком порту слушать (по умолчанию: 5555)
```

#### Примеры
```bash
# Запустить на localhost 
python main.py web --host 127.0.0.1 --port 5555

# Запустить для доступа из сети (все интерфейсы)
python main.py web --host 0.0.0.0 --port 8080

# Запустить на стандартные параметры
python main.py web
```

#### Возвращает
- Веб-приложение доступно по адресу `http://host:port/`
- Автоматически сканирует папку `results_dir`
- Показывает список обработанных писем

#### Для чего
- 🌐 Просмотр результатов анализа
- 🔍 Поиск по компаниям, контактам, продуктам
- 📊 Группировка данных
- 📥 Экспорт в Excel
- 👁️ Красивый интерфейс для нетехнических пользователей

### 2. PARSE - Парсинг из IMAP

#### Описание
Подключается к IMAP почтовому ящику, загружает письма, сохраняет их локально и анализирует содержимое.

#### Синтаксис
```bash
python main.py parse [опции]
```

#### Опции
```
--engine {local-nn|ollama|combined}
    Выбор движка анализа (по умолчанию: combined)
    - local-nn: Локальная нейросеть (быстро, без интернета)
    - ollama: Локальная LLM модель Ollama
    - combined: Ollama + fallback на local-nn при ошибке
```

#### Примеры
```bash
# Парсинг с комбинированным движком (рекомендуется)
python main.py parse

# Парсинг только с локальной нейросетью
python main.py parse --engine local-nn

# Парсинг с помощью Ollama
python main.py parse --engine ollama

# С переменной окружения
export ENGINE=local-nn
python main.py parse
```

#### Процесс
1. Подключается к IMAP серверу (из config.yaml)
2. Скачивает письма из INBOX
3. Сохраняет их в `storage.emails_dir` как .eml файлы
4. Анализирует каждое письмо выбранным движком
5. Сохраняет результаты в `storage.results_dir` как .json файлы
6. Отслеживает обработанные письма в `storage.processed_db`

#### Возвращает
- 📧 Сохраненные письма (.eml файлы)
- 📄 JSON файлы с результатами анализа
- 📋 Обновленную БД обработанных писем

#### Для чего
- 📮 Автоматический импорт писем из реальной почты
- 🤖 Полный анализ входящих писем
- 💾 Локальное хранилище письма
- 🔄 Идемпотентная обработка (письма анализируются один раз)

#### Требования
- Доступ в интернет (подключение к IMAP серверу)
- Корректные учетные данные в config.yaml
- Открытый порт 993 (обычно)

### 3. REPROCESS - Переобработка локальных писем

#### Описание
Берет сохраненные .eml файлы из папки `emails_dir` и заново анализирует их, перезаписывая результаты в папку `results_dir`.

#### Синтаксис
```bash
python main.py reprocess [опции]
```

#### Опции
```
--engine {local-nn|ollama|combined}
    Выбор движка анализа (по умолчанию: combined)
    
--only-missing
    Обновить только файлы, у которых отсутствует JSON результат
    (не трогать уже обработанные файлы)
```

#### Примеры
```bash
# Переобработать все письма с локальной нейросетью
python main.py reprocess --engine local-nn

# Переобработать только новые письма с combined движком
python main.py reprocess --only-missing

# Переобработать все письма с Ollama
python main.py reprocess --engine ollama

# Переделать анализ для всех писем новым movimento
python main.py reprocess --engine combined
```

#### Процесс
1. Сканирует папку `storage.emails_dir`
2. Находит все .eml файлы
3. Читает содержимое каждого файла
4. Анализирует текст выбранным движком
5. Сохраняет JSON результаты (перезаписывает старые)
6. Показывает прогресс обработки

#### Возвращает
- 📄 JSON файлы с результатами анализа (перезаписываются)
- 📊 Статистику обработки (кол-во файлов, ошибок, времени)

#### Для чего
- ♻️ Переанализ писем с другим движком
- 🔄 Обновление результатов после улучшений в коде
- ✅ Обработка новых писем, добавленных в папку
- 🔧 Тестирование разных движков анализа
- 📈 Повышение качества анализа (если появилась лучшая модель)

#### Практическое применение
```bash
# Сценарий 1: Обработать новые письма (очень быстро)
python main.py reprocess --only-missing

# Сценарий 2: Переделать анализ для лучшего результата
python main.py reprocess --engine combined

# Сценарий 3: Обновить только результаты, которых нет
python main.py reprocess
```

## Конфигурация

### Файл config.yaml

```yaml
# ===== IMAP НАСТРОЙКИ =====
imap:
  server: mail.nic.ru
  # Адрес IMAP сервера вашей почты
  # Примеры: 
  #   - mail.google.com (Gmail)
  #   - imap.mail.yahoo.com (Yahoo)
  #   - mail.nic.ru (Nic.ru)
  
  port: 993
  # Порт подключения
  # 993 - стандартный SSL/TLS порт
  # 143 - незащищенный порт (редко используется)
  
  username: Tech@amplifier1.ru
  # Ваша почта (как правило, полный адрес)
  
  password: 'your_password'
  # Пароль от почты
  # ⚠️ ВАЖНО: храните отдельно для production!
  
  mailbox: INBOX
  # Папка для парсинга
  # Примеры: INBOX, Sent, Archive, [Gmail]/All Mail

# ===== ХРАНИЛИЩЕ ДАННЫХ =====
storage:
  emails_dir: ./data/emails
  # Папка где хранятся скачанные письма (.eml файлы)
  # Создается автоматически если ее нет
  
  results_dir: ./data/results
  # Папка где хранятся результаты анализа (.json файлы)
  # Каждому письму соответствует JSON с извлеченными данными
  
  processed_db: ./data/processed.json
  # БД для отслеживания уже обработанных писем
  # Нужна чтобы не анализировать письмо дважды

# ===== OLLAMA (опционально) =====
ollama:
  api_url: http://localhost:11434/api/generate
  # URL API сервера Ollama (если используется)
  # Ollama должна быть установлена и запущена
  # https://ollama.ai
  
  model: mistral:7b
  # Модель Ollama для анализа
  # Другие варианты: llama2, neural-chat, etc.

# ===== ФИЛЬТРАЦИЯ СПАМА =====
spam_filters:
  enabled: true
  # Включить фильтр спама
  # Если false, все письма будут обработаны
  
  subject_blacklist:
    - unsubscribe
    - viagra
    - casino
    - lottery
    - winner
    - click here
    - limited offer
    - act now
  # Список слов которые блокируют письмо по теме
  # Регистронезависимый поиск
  
  body_blacklist:
    - unsubscribe
    - viagra
    - casino
    - cryptocurrency
    - bitcoin
    - click here now
    - limited time
    - buy now
  # Список слов которые блокируют письмо по содержимому
  # Регистронезависимый поиск
  
  sender_blacklist:
    - '*@spam.com'          # Все письма с домена spam.com
    - '*noreply*'           # Адреса содержащие noreply
    - 'notification@*'      # Адреса начинающиеся с notification@
  # Список паттернов для блокировки отправителей
  # * - любое количество символов
  
  min_company_length: 2
  # Минимальная длина названия компании
  # Письма с названиями компаний короче 2 символов будут заблокированы
  # 0 - отключить эту проверку
  
  min_email_count: 0
  # Минимальное количество email адресов в письме
  # Письма с недостаточным количеством email будут заблокированы
  # 0 - отключить эту проверку
  
  min_contact_info: 1
  # Минимальное количество контактной информации (emails + phones)
  # Письма с недостаточной контактной информацией будут заблокированы
  # 0 - отключить эту проверку
  
  exclude_replies: false
  # Блокировать письма в которых тема начинается с "Re:"
  # true - письма-ответы будут отфильтрованы
  
  exclude_forwards: false
  # Блокировать письма в которых тема начинается с "Fwd:"
  # true - пересланные письма будут отфильтрованы
  
  max_email_size_mb: -1
  # Максимальный размер письма в МБ
  # -1 - без лимита (все письма обрабатываются)
  # 10 - письма больше 10 МБ будут отфильтрованы
  
  min_email_size_kb: 0
  # Минимальный размер письма в КБ
  # 0 - нет минимума
  # 10 - письма меньше 10 КБ будут отфильтрованы
```

## Режимы обработки

### Режим 1: LOCAL-NN (локальная нейросеть)

**Что это**: Использует локальную многоязычную нейросеть (не требует интернета)

**Преимущества**:
- ✅ Результаты получаются мгновенно
- ✅ Не требует интернета
- ✅ Не требует внешних сервисов
- ✅ Быстро (13+ писем/сек)

**Недостатки**:
- ❌ Менее точная, чем LLM
- ❌ Требует более мощный ПК (GPU желателен)

**Использование**:
```bash
python main.py parse --engine local-nn
python main.py reprocess --engine local-nn
```

**Модель**: `Davlan/xlm-roberta-base-ner-hrl`  
**Язык**: Многоязычная (русский, английский и др.)

---

### Режим 2: OLLAMA (локальная LLM)

**Что это**: Использует локально установленную большую языковую модель (LLM)

**Требования**:
- Установить Ollama: https://ollama.ai
- Скачать модель: `ollama pull mistral:7b`
- Запустить сервер: `ollama serve`

**Преимущества**:
- ✅ Очень точный анализ
- ✅ Понимает контекст
- ✅ Может обрабатывать сложные письма
- ✅ Поддерживает кастомные инструкции

**Недостатки**:
- ❌ Медленнее (зависит от модели и ПК)
- ❌ Требует установку Ollama
- ❌ Требует значительно больше RAM (8GB+)

**Использование**:
```bash
# 1. Установить Ollama и модель
ollama pull mistral:7b

# 2. Запустить Ollama сервер (в отдельном терминале)
ollama serve

# 3. Использовать в программе
python main.py parse --engine ollama
python main.py reprocess --engine ollama
```

### Режим 3: COMBINED (комбинированный - рекомендуется)

**Что это**: Пробует сначала Ollama, если недоступна - автоматически переходит на local-nn

**Идеально для**:
- Если вы установили Ollama (лучший результат)
- Если Ollama по какой-то причине упадет (программа продолжит работать)
- Если вы хотите максимальной надежности

**Преимущества**:
- ✅ Лучше всего из обоих миров
- ✅ Максимальная надежность
- ✅ Если Ollama не доступна - программа не сломается

**Использование**:
```bash
python main.py parse --engine combined
python main.py reprocess --engine combined
python main.py reprocess  # По умолчанию используется combined
```

## Решение проблем

### Проблема 1: "Connection refused" при подключении к IMAP

**Причина**: Неверные учетные данные или сервер недоступен

**Решение**:
```bash
# 1. Проверьте конфиг
cat config.yaml

# 2. Может быть нужен пароль приложения (для Gmail, Яндекса и др.)
# На Gmail: https://support.google.com/accounts/answer/185833

# 3. Проверьте доступ к интернету
ping mail.nic.ru

# 4. Проверьте, открыт ли на ПК исходящий порт 993
# Иногда блокирует корпоративный firewall
```

---

### Проблема 2: "ModuleNotFoundError: No module named..."

**Причина**: Зависимость не установлена

**Решение**:
```bash
# Переустановите все зависимости
pip install -r requirements.txt --upgrade

# Или конкретный пакет
pip install <имя_пакета>

# Проверьте виртуальное окружение активировано
# Windows:
ollama_env\Scripts\activate

# Linux/Mac:
source ollama_env/bin/activate
```

---

### Проблема 3: "Port 5555 already in use"

**Причина**: Порт занят другим приложением

**Решение**:
```bash
# Используйте другой порт
python main.py web --port 8080

# Или убейте процесс на старом порту
# Windows:
netstat -ano | findstr :5555
taskkill /PID <PID> /F

# Linux/Mac:
lsof -i :5555
kill -9 <PID>
```

---

### Проблема 4: "Ollama connection refused"

**Причина**: Ollama не запущена или недоступна

**Решение**:
```bash
# 1. Запустите Ollama сервер (в другом терминале)
ollama serve

# 2. Если установить Ollama не хотите - используйте local-nn
python main.py reprocess --engine local-nn

# 3. Проверьте, что Ollama доступна
curl http://localhost:11434/api/tags
```

---

### Проблема 5: "Out of memory" или "CUDA out of memory"

**Причина**: Недостаточно памяти на ПК или GPU

**Решение**:
```bash
# 1. Обработайте меньше писем
python main.py reprocess --only-missing

# 2. Используйте local-nn вместо Ollama
python main.py reprocess --engine local-nn

# 3. Увеличьте виртуальную память Windows
# Windows: Параметры -> Система -> Дополнительные параметры системы -> 
#          Производительность -> Виртуальная память

# 4. Обновите GPU драйверы (если используете)
```

---

### Проблема 6: "SyntaxError" при запуске

**Причина**: Python версия слишком старая

**Решение**:
```bash
# Проверьте версию Python
python --version

# Должна быть 3.9 или выше
# Если старше - обновите: https://www.python.org/downloads/

# Проверьте, что используете правильный интерпретатор
which python  # Linux/Mac
where python  # Windows
```

---

### Проблема 7: "No such file or directory: config.yaml"

**Причина**: Вы не в правильной папке

**Решение**:
```bash
# Убедитесь, что вы в папке проекта
ls  # или dir в Windows
# Должны быть файлы: main.py, config.yaml, requirements.txt

# Если нет - перейдите в правильную папку
cd /path/to/email_parser20
```

## FAQ (Часто задаваемые вопросы)

### Q: Насколько быстро работает программа?
**A**: Примерно 13 писем в секунду при local-nn режиме, зависит от ПК. С Ollama медленнее, но точнее.

---

### Q: Безопасна ли программа? Где хранятся мои письма?
**A**: 
- Письма и результаты хранятся **локально** на вашем ПК
- Не отправляются никому без вашего разрешения
- Пароль от почты должен храниться безопасно (не коммитить в git!)

---

### Q: Можно ли использовать с облачными почтовыми сервисами?
**A**: Да! Поддерживает любые IMAP серверы:
- Gmail (с паролем приложения)
- Яндекс.Почта
- Mail.ru
- Собственные корпоративные серверы
- И многие другие

---

### Q: Сколько писем может обработать программа?
**A**: Теоретически - неограниченно. Практически - зависит от места на диске и памяти. На обычном ПК можно обработать 10,000+ писем без проблем.

---

### Q: Можно ли экспортировать результаты?
**A**: 
- При помощи `/export` endpoint (Excel файл)
- Прямое использование JSON файлов
- Любые скрипты могут читать результаты из папки `results_dir`

---

### Q: Какие данные извлекаются из писем?
**A**: 
- 👤 Полное имя контакта
- 📧 Email адреса
- 📞 Номера телефонов
- 🏢 Название компании
- 💼 Должность контакта
- 🛍️ Интересующие продукты

---

### Q: Почему письмо обработано неправильно?
**A**: Несколько причин:
- Письмо на английском может анализироваться менее точно
- Помните, что это __автоматический__ анализ (не 100%)
- Попробуйте режим `combined` с Ollama для лучшей точности
- Отредактируйте JSON файл вручную если нужно
